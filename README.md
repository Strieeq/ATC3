# 🔐 Atomic TriFlow Cipher v3.1

> **Современный десктопный шифратор** с симметричной криптографией, элегантным интерфейсом и опциональной привязкой к железу.  
> Разработан для безопасного хранения и передачи текстовых и бинарных данных.

## ✨ Основные возможности

*   🛡️ **Argon2id KDF** — вывод ключа из пароля, устойчивый к GPU/ASIC и side-channel атакам.
*   🔒 **ChaCha20-Poly1305** — современное AEAD-шифрование (конфиденциальность + аутентификация).
*   ✅ **BLAKE2b HMAC (64 байта)** — финальная подпись пакета, проверка целостности *до* начала расшифровки.
*   💻 **Hardware Lock v2** — опциональная привязка к уникальным параметрам железа (CPU, Motherboard, BIOS, MAC).
*   📁 **Универсальный ввод** — шифрование/дешифрование текста или файлов в один клик.
*   💾 **Нативный формат `.atc3`** — компактный бинарный формат для зашифрованных файлов.
*   🎨 **PyQt6 GUI** — адаптивный интерфейс в молочно-лиловой палитре.
*   🧩 **Модульная архитектура** — четкое разделение на ядро, воркер, UI и стили.
*   🔐 **Защита памяти** — автоматическое затирание ключей и чувствительных данных после использования.
*   🕒 **Безопасный буфер обмена** — автоматическая очистка скопированных данных через 30 секунд.

## 🔐 Криптографическая схема

### Структура пакета `.atc3`

```text
[ SIG "ATC3" ] [ VER ] [ FLAGS ] [ KDF_SALT (32B) ] [ HW_SALT (16B) ] 
[ NONCE (12B) ] [ CIPHERTEXT ] [ POLY1305_TAG (16B) ] [ HMAC_BLAKE2B (64B) ]
```

### Параметры
*   **kdf_salt**: 32 байта, случайная соль для Argon2id.
*   **hw_salt**: 16 байтов, соль для генерации отпечатка железа.
*   **nonce**: 12 байтов (8 случайных + 4 байта глобального счетчика) для ChaCha20.
*   **poly1305_tag**: 16 байтов, тег аутентификации AEAD.
*   **hmac_blake2b**: 64 байта, финальная подпись всего пакета.

### Алгоритм шифрования
1.  Генерация случайных солей и nonce.
2.  Вывод мастер-ключа из пароля через **Argon2id** (`time=3`, `mem=64MB`, `threads=4`).
3.  Если включен **HW Lock**: к материалу ключа добавляется отпечаток железа.
4.  Финальный ключ шифрования: `BLAKE2b(master_key + hw_fingerprint)[:32]`.
5.  Шифрование данных через **ChaCha20-Poly1305** (AEAD) с AAD = заголовок пакета.
6.  Подпись всего полученного пакета через **HMAC-BLAKE2b** (64 байта).

### Алгоритм дешифрования
1.  Проверка сигнатуры `ATC3` и версии.
2.  Извлечение солей, nonce, ciphertext, тега и HMAC.
3.  Вычисление HW fingerprint (если требуется флагом).
4.  Повторный вывод ключа тем же методом.
5.  **Проверка HMAC** (защита от timing attacks via `compare_digest`).
6.  Расшифровка данных через ChaCha20-Poly1305.

## 🛡️ Безопасность и защита

### Защита от атак
| Тип атаки | Мера защиты |
| :--- | :--- |
| **Timing Attacks** | Использование `hmac.compare_digest()` для сравнения тегов. |
| **GPU/ASIC Brute-force** | Argon2id с высокими параметрами памяти и времени. |
| **Side-channel / Memory Dump** | Ключи и чувствительные данные затираются в памяти сразу после использования. |
| **Error Oracle** | Единые сообщения об ошибках («Неверный ключ или поврежден файл»), не раскрывающие причину сбоя. |
| **Nonce Reuse** | Гибридный nonce: случайная часть + глобальный счетчик (пространство $2^{96}$). |

### Ограничения
*   **Симметричная криптография**: Ключ (пароль) должен передаваться получателю по защищенному каналу.
*   **Нет Forward Secrecy**: Для свойств совершенной прямой секретности используйте версию **ATC4** (с X25519).
*   **Стабильность HW Lock**: Зависит от доступности WMI (Windows) или системных утилит (Linux/Mac). Замена материнской платы или обновление BIOS может изменить отпечаток.

| **Forward Secrecy** | Да (DH) | Нет *(см. ATC4)* |

## ⚠️ Примечания по безопасности

1.  **Канал передачи ключа**: Проект использует симметричную криптографию. Никогда не передавайте пароль по открытым каналам связи.
2.  **Библиотеки**: Используются проверенные криптографические примитивы из `pycryptodome` и `argon2-cffi`.
3.  **Резервное копирование**: При использовании HW Lock настоятельно рекомендуется хранить резервную копию важных данных **без** привязки к железу.
4.  **Аудит**: Проект отлично подходит для личных архивов и обучения, но не предназначен для критических продакшен-систем без независимого криптографического аудита.

---

## 🐛 Известные ограничения

*   Шифрование очень больших файлов (>1 ГБ) может потреблять значительный объем оперативной памяти.
*   **Hardware Lock** наиболее стабилен на **Windows**. На Linux/macOS возможны нюансы с получением серийных номеров компонентов.
*   Обновление BIOS/UEFI может привести к изменению отпечатка железа.
